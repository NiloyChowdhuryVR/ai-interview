========================================================
AI INTERVIEWER - COMPLETE WORKFLOW & CODE EXECUTION PATH
========================================================

This document explains exactly how the code flows from the moment you open the website to the moment you get your final results. It is broken down step-by-step.

--------------------------------------------------------
STEP 1: The Landing Page (app/page.tsx)
--------------------------------------------------------
1. When you open http://localhost:3000/, Next.js loads `app/page.tsx`.
2. This is a very simple React component that renders the basic HTML text you see.
3. At the bottom, there is a Next.js <Link> component wrapping a button. 
4. When you click "Start Interview", it redirects your browser to the `/interview` route.

--------------------------------------------------------
STEP 2: The Interview Page Initializes (app/interview/page.tsx)
--------------------------------------------------------
1. Next.js loads `app/interview/page.tsx`.
2. At the top of this file, we call several custom "hooks" to set up our tools:
   - `useInterviewStore()`: Connects to our global state manager (Zustand) located in `store/interviewStore.ts`. This holds our transcript, questions, and current status.
   - `useSpeechRecognition()`: Connects to `hooks/useSpeechRecognition.ts`. This initializes the browser's built-in microphone listener.
   - `useSpeechSynthesis()`: Connects to `hooks/useSpeechSynthesis.ts`. This initializes the browser's built-in robot voice.
3. The page renders the initial UI with the status "idle" and shows the "Start Interview" button.

--------------------------------------------------------
STEP 3: Clicking "Start Interview"
--------------------------------------------------------
1. You click the "Start Interview" button. This triggers the `handleStartInterview` function in `app/interview/page.tsx`.
2. It calls `initInterview()` from the store. If you look in `store/interviewStore.ts`, this function grabs 8 random questions from the hardcoded list in `lib/questions.ts`.
3. It adds an introductory text to the transcript array.
4. It calls `await speak(introText)`. This triggers the code in `hooks/useSpeechSynthesis.ts` telling your browser to read the text out loud.
5. Once the robot finishes speaking, it calls `askQuestion(0)`.

--------------------------------------------------------
STEP 4: Asking a Question
--------------------------------------------------------
1. The `askQuestion(index)` function (in `app/interview/page.tsx`) reads the current question from the store.
2. It updates the state status to 'asking' and adds the question text to the transcript.
3. It calls `await speak(question.text)` so the AI reads the question out loud.
4. Once done speaking, it sets the status to 'listening'.
5. It calls `startListening()` from the `useSpeechRecognition.ts` hook. This turns on your microphone.

--------------------------------------------------------
STEP 5: You Answer the Question
--------------------------------------------------------
1. As you speak, `hooks/useSpeechRecognition.ts` receives text chunks from the Google Chrome speech engine.
2. It updates a variable called `speechTranscript` in real-time.
3. You see your words appear on the screen under "Your Voice (Mic)".
4. When you click "Submit Answer", it triggers the `submitAnswer` function in `app/interview/page.tsx`.
5. It immediately turns off the microphone by calling `stopListening()`.
6. It adds your finalized answer to the transcript array.
7. It changes the status to 'evaluating'.

--------------------------------------------------------
STEP 6: The Backend Evaluates Your Answer
--------------------------------------------------------
1. Inside `submitAnswer`, the code makes an HTTP POST request to `/api/evaluate`.
2. This request goes to the backend file: `app/api/evaluate/route.ts`.
3. The backend receives your answer and the original question.
4. It calls `buildEvaluatePrompt()` (from `lib/gemini.ts`) to create a specific prompt instructing the AI how to grade you.
5. It sends this prompt to the Google Gemini AI API.
6. Gemini replies with a JSON object containing a score (1-10) and text feedback.
7. The backend sends this JSON back to the frontend.
8. The frontend receives the grade, saves it to the store via `addEvaluation()`, and makes the robot speak a quick acknowledgment ("Thank you for your answer...").
9. Finally, it checks if there are more questions. If yes, it calls `askQuestion(nextIndex)` (Go back to Step 4). If no, it calls `handleEndInterview()`.

--------------------------------------------------------
STEP 7: Ending the Interview
--------------------------------------------------------
1. The `handleEndInterview` function (in `app/interview/page.tsx`) runs.
2. It makes the robot speak a goodbye message.
3. It makes another HTTP POST request, this time to `/api/feedback`.
4. This goes to the backend file: `app/api/feedback/route.ts`.
5. The backend takes ALL your question evaluations, calculates an overall score, and asks Gemini to generate a final summary and list of strengths/improvements.
6. It sends this final report back to the frontend.
7. The frontend saves this entire report into your browser's memory (`sessionStorage`).
8. It triggers `router.push('/dashboard')` which redirects your browser to the dashboard page.

--------------------------------------------------------
STEP 8: The Dashboard (app/dashboard/page.tsx)
--------------------------------------------------------
1. Next.js loads `app/dashboard/page.tsx`.
2. As soon as the page loads, the `useEffect` hook runs.
3. It reads the feedback report from `sessionStorage` (which we saved in Step 7).
4. It saves this report into a local React state variable `const [feedback, setFeedback] = useState(...)`.
5. The plain HTML maps over this data to display your Overall Score, Strengths, Improvements, and a list of all your Questions with their individual scores.
6. If you click "Try Another Interview", it just links you back to `/interview`, starting the cycle over again.

========================================================
KEY FILES TO STUDY:
- `app/interview/page.tsx` (The main controller for the whole process)
- `store/interviewStore.ts` (Where all the data like transcripts and scores live during the interview)
- `hooks/useSpeechRecognition.ts` (How the microphone code works)
- `app/api/evaluate/route.ts` (How the Next.js backend talks to the Google Gemini AI)
